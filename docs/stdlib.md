# JVAV Standard Library

The JVAV standard library (`std/`) provides common utilities designed to work within the constraints of the JVL language and the MimiWorld ownership system. It is intentionally minimal — no complex data structures, no garbage collection, and no hidden allocations.

## Design Philosophy

- **Zero overhead** — Pure JVL functions; no compiler magic or intrinsic special-casing
- **Ownership-safe** — All APIs respect MimiWorld move/borrow rules
- **Word-addressable** — Memory operations operate on 128-bit words, matching the VM
- **No hidden I/O** — Only `std/io.jvl` performs output; other modules are pure computation

## Import Mechanism

Standard library modules live in the `std/` directory at the **project root**. jvlc automatically searches this directory regardless of where your source file is located.

```jvl
import "std/io.jvl";
import "std/math.jvl";
```

Relative imports (e.g. `import "./lib/util.jvl";`) resolve from the source file's directory. The `std/` path is a special fallback resolved from the executable's project root.

## Modules

### `std/io.jvl` — Console Output

Built on top of the VM's memory-mapped I/O ports (`0xFFF0` putchar, `0xFFF1` getchar, `0xFFF2` putint, `0xFFF3` getint, `0xFFF4` puthex).

| Function | Description | Example Output |
|----------|-------------|----------------|
| `putint(x)` | *Built-in.* Print integer to stdout | `42` |
| `putchar(c)` | *Built-in.* Print ASCII character | `A` |
| `println(x)` | Print integer + newline | `42\n` |
| `print_newline()` | Print `\n` | `\n` |
| `print_space()` | Print ` ` | ` ` |

```jvl
import "std/io.jvl";

func main(): int {
    println(42);
    println(7);
    return 0;
}
// Output: 42\n7\n
```

### `std/math.jvl` — Integer Math

All functions operate on signed 128-bit integers. No floating-point support.

| Function | Description |
|----------|-------------|
| `abs(x)` | Absolute value |
| `max(a, b)` | Greater of two integers |
| `min(a, b)` | Lesser of two integers |
| `clamp(x, lo, hi)` | Restrict `x` to `[lo, hi]` |
| `pow(base, exp)` | Integer exponentiation (`base^ᵇ`, `exp ≥ 0`) |

```jvl
import "std/math.jvl";

func main(): int {
    println(abs(-5));        // 5
    println(clamp(10, 0, 5)); // 5
    println(pow(2, 8));      // 256
    return 0;
}
```

**Note:** `pow` uses iterative multiplication. Large exponents will overflow the 128-bit range silently (wrap-around behavior depends on VM implementation).

### `std/mem.jvl` — Memory Operations

JVAV is **word-addressable** — each address points to one 128-bit word, not one byte. `memcpy` and `memset` operate in units of words.

| Function | Description |
|----------|-------------|
| `memcpy(dst, src, n)` | Copy `n` words from `src` to `dst` |
| `memset(p, val, n)` | Set `n` words starting at `p` to `val` |

```jvl
import "std/mem.jvl";

func main(): int {
    var a: ptr<int> = alloc(3);
    a[0] = 1; a[1] = 2; a[2] = 3;

    var b: ptr<int> = alloc(3);
    memcpy(b, a, 3);

    memset(a, 0, 3);   // a[0..2] = 0

    free(a);
    free(b);
    return 0;
}
```

**Ownership interaction:** Both `dst` and `src` are borrowed (not moved) because `ptr<T>` arguments in JVL are passed by value — the pointer itself is copied, the pointed-to memory is not.

### `std/string.jvl` — String Output

JVL string literals (`"hello"`) are stored as contiguous word arrays in the data segment. They are **not null-terminated**; you must track length manually.

| Function | Description |
|----------|-------------|
| `str_putn(s, n)` | Print first `n` characters of string `s` |

```jvl
import "std/string.jvl";

func main(): int {
    var msg = "JVAV";
    str_putn(msg, 4);   // prints: JVAV
    return 0;
}
```

**Important:** `s[i]` on a `ptr<char>` returns the character at word offset `i`. Because JVAV is word-addressable, each character occupies one full word. This matches the `DB "..."` layout generated by the backend.

## Interaction with MimiWorld Ownership

The standard library is fully compatible with MimiWorld rules:

- **Copy types** (`int`, `char`, `bool`) — passed by value, no ownership transfer
- **Move types** (`ptr<T>`) — the pointer is copied, but the allocation it points to is *not* moved
- **No hidden allocations** — `std/io` and `std/math` never call `alloc()`; `std/mem` and `std/string` only operate on pointers you provide

```jvl
import "std/mem.jvl";

func dup3(src: ptr<int>): ptr<int> {
    var dst = alloc(3);
    memcpy(dst, src, 3);   // src is not moved; only read through borrow
    return dst;             // caller owns dst
}

func main(): int {
    var a = alloc(3);
    a[0] = 7; a[1] = 8; a[2] = 9;

    var b = dup3(a);        // a is still valid here
    free(a);
    free(b);
    return 0;
}
```

## Writing Custom Library Modules

Any `.jvl` file can be imported. Follow these conventions for consistency:

1. **No `main()` function** in library files — libraries export functions, not entry points
2. **Explicit parameter types** — JVL requires annotated function parameters
3. **Document ownership** — if your function consumes a pointer (e.g. wrapper around `free`), name it clearly
4. **Place in `std/` or a project-local directory** — use `import "std/foo.jvl";` or `import "lib/foo.jvl";`

```jvl
// lib/utils.jvl
func square(x: int): int {
    return x * x;
}
```

```jvl
// main.jvl
import "lib/utils.jvl";

func main(): int {
    println(square(7));   // 49
    return 0;
}
```
