# System Call Reference

This document provides detailed reference for all JVAV system calls. Syscalls are invoked via the mailbox at `0xFFE0`–`0xFFE4`, or through `.syscall` wrapper functions generated by `jvavc`.

## File & Memory Mapping

### SYS_MMAP_FILE (1)

Memory-map a host file into the VM address space.

**Arguments:**
- `ARG0`: Address of filename string (word array, one char per word)
- `ARG1`: VM address to map the file to
- `ARG2`: Size in words to map

**Return:** Handle ID (>0) or 0 on failure.

**Assembly example:**
```asm
    LDI R0, fname
    LDI R1, 0x1000
    LDI R2, 256
    PUSH R2
    PUSH R1
    PUSH R0
    CALL mmap_file
    LDI R4, 3
    ADD SP, SP, R4
    ; R0 now contains handle or 0
    .syscall mmap_file, 1, 3
```

**Behavior:** Opens the file in read/update mode. Read/write to mapped VM addresses transparently reads/writes the file. The mapping remains active until `SYS_MUNMAP`.

---

### SYS_MUNMAP (2)

Unmap a previously memory-mapped file.

**Arguments:**
- `ARG0`: Handle ID returned by `SYS_MMAP_FILE`

**Return:** `0` on success, `-1` on error.

---

### SYS_MSYNC (3)

Flush a memory-mapped file to disk.

**Arguments:**
- `ARG0`: Handle ID

**Return:** `0` on success, `-1` on error.

---

## File I/O

### SYS_FOPEN (4)

Open a host file via C `fopen()`.

**Arguments:**
- `ARG0`: Address of filename string
- `ARG1`: Address of mode string (e.g., `"r"`, `"w"`, `"rb"`)

**Return:** File descriptor (>0) or 0 on failure.

**Assembly example:**
```asm
    LDI R0, fname
    LDI R1, mode
    PUSH R1
    PUSH R0
    CALL fopen
    LDI R4, 2
    ADD SP, SP, R4
    .syscall fopen, 4, 2

fname: DB "data.txt"
mode:  DB "rb"
```

**Notes:**
- FDs are 1-based internally (FD 0 means error)
- Maximum open files: `MAX_FD` (typically 16)

---

### SYS_FCLOSE (5)

Close an open file descriptor.

**Arguments:**
- `ARG0`: File descriptor

**Return:** `0` on success, `-1` on error.

---

### SYS_FREAD (6)

Read bytes from a file into VM memory.

**Arguments:**
- `ARG0`: File descriptor
- `ARG1`: VM buffer address
- `ARG2`: Number of bytes to read

**Return:** Number of bytes actually read, or `-1` on error.

**Behavior:** Reads byte-by-byte. Each byte is stored in the low byte of a word. Does **not** pack multiple bytes into a single word.

---

### SYS_FWRITE (7)

Write bytes from VM memory to a file.

**Arguments:**
- `ARG0`: File descriptor
- `ARG1`: VM buffer address
- `ARG2`: Number of bytes to write

**Return:** Number of bytes actually written, or `-1` on error.

**Behavior:** Writes the low byte of each word. Automatically flushes after writing.

---

### SYS_FSEEK (8)

Seek to a position in a file.

**Arguments:**
- `ARG0`: File descriptor
- `ARG1`: Offset
- `ARG2`: Whence (`SEEK_SET=0`, `SEEK_CUR=1`, `SEEK_END=2`)

**Return:** New position, or `-1` on error.

---

### SYS_FTELL (9)

Get current file position.

**Arguments:**
- `ARG0`: File descriptor

**Return:** Current position, or `-1` on error.

---

## Memory Operations

### SYS_MEMCPY (10)

Copy words from one VM memory region to another.

**Arguments:**
- `ARG0`: Destination address
- `ARG1`: Source address
- `ARG2`: Count in words

**Return:** `0`

**Behavior:** Copies word-by-word. Source and destination may overlap (copy is done in reverse order internally).

---

### SYS_MEMSET (11)

Set a region of VM memory to a value.

**Arguments:**
- `ARG0`: Destination address
- `ARG1`: Value to write
- `ARG2`: Count in words

**Return:** `0`

---

## Heap Allocation

### SYS_MALLOC (12)

Allocate heap memory.

**Arguments:**
- `ARG0`: Number of 128-bit words to allocate

**Return:** Address of allocated block, or `0` on failure.

**Frontend:** `alloc(n)`

**Behavior:** Simple bump allocator. Returns the current `heap_ptr` and advances it by `size`. Attempts to grow VM memory if the allocation would collide with the stack.

---

### SYS_FREE (13)

Free a heap allocation.

**Arguments:**
- `ARG0`: Address returned by `SYS_MALLOC`

**Return:** `0` on success, `-1` on error.

**Frontend:** `free(p)`

**Behavior:** Writes a tombstone value (`0xDEAD`) to the first word of the block. Does not reclaim memory (bump allocator).

---

## Console I/O

### SYS_PUTCHAR (14)

Print a single ASCII character to stdout.

**Arguments:**
- `ARG0`: Character code (only low byte is used)

**Return:** `0`

**Frontend:** `putchar(c)`

**Example:**
```jvl
putchar(72);   // 'H'
putchar(105);  // 'i'
putchar(33);   // '!'
```

---

### SYS_PUTINT (15)

Print a signed integer to stdout in decimal.

**Arguments:**
- `ARG0`: Integer value (full 128-bit range)

**Return:** `0`

**Frontend:** `putint(x)`

**Example:**
```jvl
putint(42);     // prints "42"
putint(-99);    // prints "-99"
putint(0);      // prints "0"
```

**Note:** Does not print a newline. Use `putchar(10)` or `print_newline()` for that.

---

### SYS_GETCHAR (16)

Read a single character from stdin.

**Arguments:** None.

**Return:** Character code, or `-1` on EOF.

**Note:** Not currently exposed as a JVL builtin. Use in assembly or declare as `extern`.

---

### SYS_GETINT (17)

Read a signed integer from stdin.

**Arguments:** None.

**Return:** Integer value.

**Note:** Uses C `scanf("%lld", ...)`. Not currently exposed as a JVL builtin.

---

## Process Control

### SYS_EXIT (18)

Terminate the VM with a specific exit code.

**Arguments:**
- `ARG0`: Exit code to return to the host OS

**Return:** `0` (but VM stops before return is meaningful)

**Frontend:** `exit(code)`

**Behavior:** Sets `vm->exit_code = ARG0`, sets `vm->running = 0`, and returns from the syscall wrapper. The VM halts when control returns to the execution loop. The `jvm` executable then returns this code via `exit()`.

**Example:**
```jvl
func main(): int {
    if (error_condition) {
        exit(1);
    }
    exit(0);
    return 0;
}
```

---

### SYS_PUTSTR (19)

Print a string from VM memory.

**Arguments:**
- `ARG0`: Starting address of string (word array)
- `ARG1`: Number of characters to print

**Return:** `0`

**Frontend:** `putstr(s, n)`

**Behavior:** Reads `len` consecutive words starting at `addr`, extracts the low byte of each, and prints them to stdout. Does **not** interpret null terminators — always uses explicit length.

**Assembly example:**
```asm
    LDI R0, msg
    LDI R1, 5
    PUSH R1
    PUSH R0
    CALL putstr
    LDI R4, 2
    ADD SP, SP, R4
    HALT

msg: DB "Hello"
    .syscall putstr, 19, 2
```

**JVL example:**
```jvl
var msg = "Hello, World!";
putstr(msg, 13);
```

## Error Handling

All syscalls return error indicators through `SYSCALL_RET` (`0xFFE4`):

- **File operations:** Return `0` or `-1` on error
- **Allocation:** Return `0` (null) on failure
- **Unknown command:** Returns `-1`

The VM does **not** set `errno` or provide detailed error messages for syscalls. Check the return value and handle failures appropriately.

---

### SYS_SLEEP (20)

Pause execution for a specified duration.

**Arguments:**
- `ARG0`: Duration in milliseconds

**Return:** `0`

**Frontend:** `sleep(ms)`

**Behavior:** Blocks the VM execution loop for the requested duration. On Windows uses `Sleep()`; on POSIX uses `usleep()`. Negative values are clamped to 0.

**JVL example:**
```jvl
sleep(500);   // pause for 500ms
```

## Adding Custom Syscalls

To add a new syscall to the VM:

1. Add `SYS_FOO` to the enum in `jvm/include/jvm.h`
2. Add a `case SYS_FOO:` handler in `jvm/src/jvm.c`
3. Optionally register it as a JVL builtin in `jvavc/front/src/sema.cpp`
4. Emit the `.syscall` wrapper in `jvavc/front/src/codegen.cpp`

Alternatively, for user-defined syscalls that the VM already supports, use the frontend `syscall` declaration (see `frontend_syntax.md`) without modifying compiler source code.
5. Add tests and documentation
